關於雜湊表

資料結構中有一種組織資料的方式稱為「雜湊表」，在 C# 語言中 HashSet<T>、Dictionary<TKey, TValue>、…等集合種類就是用雜湊表實現的。

可以把雜湊表想像成很多個桶子，每個桶子都有唯一的識別編號，而且每個桶子都可放入不止一件物品。

這種資料結構的優點是，當我們知道物品所在桶子的編號，只要先找到指定編號的桶子，再到這個桶子裡去找，一定會找到我們要的東西，找東西的速度會很快。比起每個桶子都打開來找，會省下不少時間。

雜湊表的概念中，把每個物品 平均分配到不同編號 的桶子是很重要的，如果把所有東西通通放進同一個編號的桶子，找東西時打開這個桶子，還要巡過每一個物品才能確定找到要的東西，那用雜湊表就沒有意義了，不如放到陣列或 List 裡。

雜湊表的概念在現實生活中的應用：圖書館的書本分類存放方式、書本的目錄、字典的索引頁。

覆寫 GetHashCode 方法

GetHashCode 就是產生雜湊編號的方法，得到編號後，在雜湊表中將這個物件放到這個編號的桶子裡，日後只要先呼叫物件的 GetHashCode 得到編號，再到雜湊表裡去找出桶子，從桶子裡取出得到這個物件，速度就快了。

C# 用雜湊表實現的集合類別

在 C# 語言中，用雜湊表實現的集合類別有很多種，例如 HashSet<T>、Dictionary<TKey, TValue>、SortedList
<TKey, TValue>、HashTable、SortedList、…等。

我們用 Data 類別建立 data1、data2 兩個物件，並將這兩個物件加入到 HashSet<T> 內，試試以下不同情境，看看會發生什麼事。

  • 雜湊碼用 Object 的 GetHashCode 方法產生。
    data1、data2 兩個物件的 GetHashCode 都不同，因此兩個物件都會加入到 HashSet<Data> 中。

  • 雜湊碼用自訂的 GetHashCode 方法產生，條件是用 Id + Name 欄位的值產生。
    data1、data2 雖然是兩個執行個體，但 GetHashCode 的值是相同的，在加入到雜湊表集合時，會先呼叫 GetHashCode 並檢查雜湊表內是否存已存在這組雜湊碼。

    • 若雜湊碼不存在，將物件加入雜湊表。
    • 若雜湊碼已存在，另外呼叫 Equals 方法比對既有物件與新加入物件是否相同，當回傳為 False 才加入雜湊表。
      此例只有 data1 會加入到 hashSet 中。

什麼情況會遇到 GetHashCode 相同，但 Equals 為 False 的情況？

當 GetHashCode、Equals 兩個方法使用不同欄位條件時，就會遇到這種情境。

如下圖，GetHashCode 只用 Id 欄位去產生，但 Equals 用 Id + Name 欄位的值比對，此時三個 Data 物件都會加入到雜湊表集合內，並且都放在同一個雜湊碼的桶子裡。

我們在 Equals 和 GetHashCode 方法內加入 Console.WriteLine() 觀察程式執行的流程，可看到雜湊表取資料，遇到相同 GetHashCode 時，會取出同一個桶子內的所有資料，再逐一呼叫 Equals 比對結果，最後才把比對吻合的那筆資料出來。
